How can you use the staging area to make sure you have one commit per logical change?

Only "git add" a file to move it to the staging area when you are ready to commit it.

What happens when you initialize a repository? Why do you need to do it?
Git creates a list of the files and keeps track of which ones have no commit.

How is the staging area different from the working directory and the repository? What value do you think it offers?  The staging area differentiates between those files that form part of a repository and those that do not.  The working directory merely shows all the files in the directory that are not hidden
